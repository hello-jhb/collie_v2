# Part II — `investment_intel.py` (Layer 3)  ·  f171002

The anti-filler design is the same trick the rest of the engine already trusts —
**code computes, GPT explains:**
- model_brief: GPT reads → code grounds (trust_engine).
- trust_engine: code computes identities → GPT challenges.
- **investment_intel: code computes the analytical bridges → GPT narrates them.**

## Deterministic analytics (`compute_analytics`, no LLM)
Pure arithmetic over **verified facts only** (`verdict == show`); each block
guarded so a missing input never kills the rest. Missing pricing legs are derived
from the cap identity (going-in NOI = price × cap):
- **Cap spread** = exit − going-in (bps): expansion = conservative, compression = aggressive.
- **NOI bridge** = exit/going-in − 1, plus CAGR over hold.
- **Value-creation split** = ΔV decomposed into operations `(N1−N0)/c0` vs
  revaluation `N1·(1/c1−1/c0)`. Tells you if the gain is *earned* or *repriced*.
- **Leverage accretion** = levered IRR − unlevered IRR (pts).
- **Development spread** = yield-on-cost − exit cap (the create-vs-buy margin).
- **Basis per unit/key.**
- **Fragility flags**: floating-rate, LTV > 70% / LTC > 75%, DSCR < 1.25×,
  debt-yield < 8%, cash-on-cash < 5%, exit-dependent (revaluation > 50% of gain).
  Thresholds are institutional DEFAULTS at the top of the module — tune to taste.

## Grounded interpretation (`interpret`, 1× gpt-4o, temp 0.2)
Handed the computed deltas + the verified facts; **forbidden to invent or
recompute** — every adjective must trace to an analytic. System prompt distilled
from the guideline's Layer 3 + judgment rules + good/bad examples. No-key →
`_deterministic_view` renders the analytics as prose, so Layer 3 always produces
something and is fully testable headless. Output is the **Initial View**.
