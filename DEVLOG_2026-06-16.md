# Part I — The universal cash-flow spine  ·  3a46f3e, 777c028

The one fact true of every DCF model: it states an IRR somewhere, and that IRR is
produced by a dated cash-flow stream (outflow → inflows). So:

**`cashflow_spine.py`** — find the engine with one fingerprint, no sheet
names / labels / roles / units:
1. find every cell that states an IRR (label-anchored),
2. find every dated, sign-changing numeric series, in **both** orientations
   (periods across columns OR down rows),
3. recompute XIRR on each,
4. the stream whose XIRR **reproduces a stated IRR** IS the engine; its sheet is
   the anchor. `ok=True` only if a stream validates.

Scale-/label-/role-/layout-invariant. **Validated across the whole corpus by
self-consistency** (each file's own stated IRR is the ground truth): 8/10 readable
models reproduced their stated IRR, mostly to 0 bps — BAC's levered stream on
`Cash Flows`, unlevered on `Model`; 1425 on `Monthly-CF`; St Regis, Westview,
Village, 1 South Beach, MSD. Honest misses: 425 Colorado (no cached values),
Outlets (equity stream isn't one dated row). Handles datetime, year-int, and
**text** ("Jul-2023") date axes.

**`workbook_map.py`** (Slice 1) — sheet roles (via orientation), economic blocks,
**formula provenance** (a summary cell `=Monthly-CF!E140` is *display*; the
formula source is truth), candidate facts per concept with value-domain gating.

# Part II — `deal_truth.py`: canonical reconstruction  ·  3a46f3e, 42aa745

Consumes the map + spine; turns candidates into ONE canonical deal truth (no GPT):

- **Cash-flow oracle** — bind levered/unlevered by recomputing XIRR/EM and
  matching the stated IRR ("the model grades the parse" — it picks the right
  stream even when its label is wrong: BAC's true levered stream is
  `Equity Net Cash Flow`, not the row labeled "Leveraged Cash Flows").
- **Rule 8 — derive the stack FROM the stream**, not vocab cells: equity = Σ
  levered outflows, total cost = Σ unlevered outflows, sale = terminal inflow,
  **acquisition = the largest single outflow** (1b89870 — fixed BAC's "$1.3M
  purchase," a closing-cost line, → $195.7M).
- **Identity checks**: debt+equity≈cost, exit≈NOI/cap, recomputed≈stated, cost≈CF
  draws, acq≈initial outflow.
- **Dynamic guardrails** — generated only from detected evidence (source conflict,
  display-vs-source, cash-flow-total-not-a-price, non-standard multiple, deal
  type, etc.); the contract a later GPT must obey.
- **Rule 9** — no validated stream → explicit `engine_not_found` (5592a1f); never
  claims "reconstructed."
- Hold from the **actual sale event** (month 60 of an 81-month model, not the
  model length). Conflict detection ignores part-of-whole (equity = LP + GP)
  (f88e055).

Wired into the app, the GPT brief (b902252), and Layer 3 (d45be05) — all read the
validated spine + obey the guardrails.

# Part III — `cashflow_rollup.py`: read the model in full  ·  b416de8, ab69578

Asking GPT to read NOI off a monthly tab fails — it's rendered truncated (first
~60 cols, char-capped), so GPT fabricates cell refs and the trust engine omits
everything ("27M/32M/34M not grounded"). Instead: read the financial-model sheets
**in full**, detect the period axis, and **roll every line item up to annual**
deterministically. The rolled-up table is grounded, loses nothing to truncation,
and is small enough to hand GPT to narrate. The operating statement is anchored to
the sheet where NOI lives — `revenue − opex = NOI` checks out (Outlets / Village /
Westview). Reads NOI correctly across the corpus where GPT hallucinated.

# Part IV — Absolute units, the right way  ·  aac4b22

Two failed attempts first (sale×cap anchor, debt×0.1 anchor) — both broke on
*some* model, because **every dollar fact can be off-scale** (debt is NOT reliably
full dollars: Westview/Village debt are in $000s; St Regis is mixed). A model
entirely in $000s is internally consistent, so absolute scale needs the model's
**own declared units** — and RE models state them: `"$ in 000s"`,
`"($ in thousands)"`. `cashflow_spine.sheet_scale` reads that header and scales
each sheet to full dollars (applied in `find_streams` so the spine/stack are
full-$, and in the roll-up so NOI is full-$; XIRR is scale-invariant, so returns
are untouched). Gaps: vocab cost/debt brought onto the stream-derived equity's
scale via the scale-invariant LTC ratio; the operating statement reconciled to
the now-correct sale (`NOI/exit_cap ≈ sale`) for the few sheets that don't declare
units. **Full-$ across the corpus:** BAC $211.6M cost / $15.8M NOI, Westview
$59.1M (was $59K), Village $98M (was $98K), St Regis $371.8M / $27M NOI.

  Profile · Cash Flow/NOI · CapEx) from the spine + roll-up; deterministic, always

  a `summary_mismatch` guardrail; **engine wins**. On BAC: acquisition engine
  $195.7M ✓ vs summary $192.0M (`General Information!C11`).
- **Lead with the analysis** (c8a802e) — render the grounded analysis first, Layer
  3 after, GPT brief tucked into a collapsed "Narrative read" expander. The
  analysis is now fully deterministic, so it renders without an API key.
