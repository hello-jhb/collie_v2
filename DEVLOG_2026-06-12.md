# Part I — Workbook Orientation (new first stage)  ·  331b2b9

## Trigger
The user kept asking why "identify the workbook" never changed extraction. The
answer: sheet identification existed (`sheet_classifier`, name tiers) but it
only ever fed a **candidate ranker** — it decided which of two NOI cells wins,
never which sheet to *read*. Every path led to "find metric X"; none led to
"understand sheet Y."

## Build
`workbook_orientation.py` — deterministic, read-only, no GPT. Classifies every
sheet's role (summary / inputs / returns / model / support) from CONTENT and
STRUCTURE:
- keyword concentration (acquisition / returns / debt / identity vocab)
- hardcode ratio (inputs are typed-in, not formulas)
- internal-formula density vs cross-sheet pulls (a summary PULLS values; an
  inputs sheet is REFERENCED BY others)
- time-series shape (model tabs carry wide dated roll-ups)

Sheet names are only a weak score bonus. Output is a workbook map; tier 99
name-skips (sensitivity/comps/backup) stay skipped regardless of content
(sensitivity grids LOOK like returns sheets — that's the trap).

Feeds the existing machinery via `orientation_tier_map()` → the `{sheet: tier}`
dict `extract_aam` / `scan_workbook_for_candidates` already consume. Visible
🧭 panel in the left column (roles, confidence, signals) so the user can verify
the engine's read before extraction.

## Validation (4 real models, deterministic)
425 Colorado + Retail byte-identical to baseline; BAC + St Regis strictly
improved. St Regis Interest Rate Spread 5.76% (stray One-Pager cell) → **3.20%
(Debt Information)** matching the chat read's "LIBOR + 320bps"; rate cap 2.50%
and LTV 69.73% recovered; a garbage exit cap (2,238,921×) eliminated. Key-sheet
role checks 9/10.

# Part VII — The comprehension-first pivot + `model_brief.py` (step 1)  ·  dde6147

## User reframe
> "This is the result I want from any excel model … without going through the
> validation process (validation works, so no human confirm). Quick, no wrong
> data, focused. Work like Claude reads Excel."

The gap to `Outcome.rtf` is NOT a validation gap. The checklist-and-gate **is**
the architecture; the Outcome is a document read. Half its richness (Fund,
Strategy, WDP, Peak Equity, Yield-on-Cost, Model Structure) is outside our 22
fields. The fix is to make comprehension the product and turn validation from a
human GATE into a silent per-number CONFIDENCE SCORE.

## Target pipeline
```text
Upload → Orient(cached) → Read key tabs whole → Comprehend(facts+brief, 1 call)
       → Trust engine → Render: show / flag / omit → Deep dives on demand
```
Trust engine = 5 signals replacing the human: **grounded** (cell is real),
**authoritative** (oriented summary/inputs/returns tab), **reconciles** (Price ×
Cap ≈ NOI, Debt + Equity ≈ Basis, …), **corroborated** (agrees across summary
tabs), **challenged** (a second read told to find a *better/different* value —
the model argues with itself). Composed figures (per-key, totals) are recomputed
deterministically. HIGH → show plain; MED → inline flag; LOW/conflict → omit.

## Built today — step 1 only (comprehension, NO trust yet)
`model_brief.py`: orient → `analyst_reading_stack` → ONE gpt-4o call returning
`{identity, facts[cell-cited], brief{overview,key_stats,debt,returns,
model_structure}}`. Disk-cached by file hash + version + model. Rendered in the
Outcome column ABOVE the audit gate, read-only — the gate is untouched. No-key
guard keeps the gate working without the LLM.

Deliberately comprehension-only so we can judge raw read quality against the
reference Outcome **before** building the trust engine that earns removing the
gate.

Mocked-GPT validation: brief assembles into the Outcome shape; reads the right
tabs (Executive Summary / One-Pager / General Information on St Regis — the chat
read's list); cache round-trips; no-key guard clean; app boots.

## Why a new repo
The pivot moves away from the gated philosophy the production engine is built
on. **collie_v2** is a full fork (complete history) so the rebuild is isolated;
collie_v1 stays the working, gated engine. Pushed public to
github.com/hello-jhb/collie_v2.

# Part VIII — Trust engine + gate demotion  ·  0c1c756, (this commit)

## Trust engine (`trust_engine.py`) — what replaces the human
Each fact the brief cites earns a verdict from five signals: **grounded** (the
cited cell really holds the value — anti-fabrication floor; read-only
`values_only` scan with computed coordinates, after the `EmptyCell.coordinate`
bug), **authoritative** (oriented summary/inputs/returns tab), **reconciles**
(satisfies a deal identity it joins — Price×Cap=NOI, Debt+Equity=Basis,
LTV=Debt/Price, Exit Value×Exit Cap=Exit NOI, over normalized display
magnitudes), and **challenged** (one adversarial gpt-4o re-read that
corroborates or finds a more-authoritative conflicting cell).

Verdict: not grounded → **omit**; grounded + (reconcile-fail | challenge-
disagree) → **flag**; grounded + authoritative + (reconcile-pass | challenge-
agree) → **high**; else grounded → **medium**. Validated headless on all five
paths, incl. the BAC $192M-vs-$88M-slice flag.

## Finalize + demote (`model_brief.finalize_brief`, app)
- `finalize_brief` regenerates the brief narrative from VERIFIED facts only —
  omitted facts never appear, flagged ones are marked "(unverified)". One
  gpt-4o call; deterministic fallback appends a flagged/omitted note. This is
  the "no wrong data" guarantee for what the user reads.
- **Gate demoted**: the Deal Brief + Fact Check is now the pre-analysis hero;
  the 22-field editable checklist moved into a collapsed optional expander (its
  body still executes so the records exist). "Proceed — confirm facts" became
  "Run analysis →", always enabled — no mandatory human confirm. The report now
  leads with the comprehension brief.

The existing confirm→ingest→snapshot→report plumbing is unchanged; the brief
+ trust layer sits in front of it.

# Part X — Live QC and the fixes it drove  ·  a4d3954, e4d0738, 433da85, 122fdde, bb24358

First deployed QC on Streamlit Cloud (St Regis logs + a South Beach run).

## What worked
- The **Deal Brief reads clean** — Outcome-quality on South Beach (dev deal,
  Miami Beach, 425 keys, purchase/dev-cost/uses, floating rate, IRR, EM, model
  structure), every figure cell-cited. The user signed off: "deal brief is good."
- **Grounding bites:** the focused gap-fill rejected an off-scope hit
  (`exit_noi` from a non-whitelisted cell), and the Fact Check correctly
  rejected fabrications.

## What broke / was wrong, and the fix
1. **Ephemeral FS crash (`a4d3954`).** A mid-run redeploy wiped `uploads/`; the
   uploader only wrote on a NEW upload, so `file_sha256` crashed mid-ingest
   (`FileNotFoundError`). Fix: always re-persist any uploaded file whose disk
   copy is missing. (Also: that crash log was the OLD pre-`977d4ab` build —
   the 8.7-min legacy ingest no longer runs on the brief path.)
2. **Grounding omitted real facts (`e4d0738`).** South Beach dropped Hotel Exit
   Value \$476.4M and Debt at Exit −\$250M as "not grounded" — but both had
   `challenge: agree`. gpt-4o cited a slightly-wrong cell for a real value. Fix:
   (a) scan the cited sheet for the value and CORRECT the provenance; (b) if
   still unmatched but the challenge agrees, show as MEDIUM ("cell unconfirmed")
   instead of omitting. Anti-fabrication floor intact.
3. **Deep dives were the wrong thing entirely (`433da85`, `122fdde`, `bb24358`).**
   The St Regis cash-flow dive returned "\$864 going-in NOI" (one MONTHLY cell
   off a proforma in a \$000s model) and was slow — it ran the old agentic
   re-read + a time-series scrape. Rebuilt as a **focused dive**: pick the
   topic's sheets (cash flow → NOI/Proforma/P&L/Cash Flows; leverage → the Debt
   tabs; returns → IRR Tracker; capex → FF&E/Capex; risks → Recession/
   Sensitivity/Scenarios), read them WHOLE, summarize in one pass with an
   "annual not monthly, mind \$000s" instruction. Sheet selection hardened:
   short keywords match whole tokens (killed "ti" → "valuaTIon"), concise
   primary sheets beat secondary/variant ones (main "P&L" beats "Golf P&L").
   Dive guarded so it can never red-screen (surfaces the error + traceback).
4. **Dives had no credibility (`bb24358`).** A dive was a bare GPT summary. Now
   it runs the SAME trust chain as the brief: GPT returns {facts, narrative};
   facts are grounded / reconciled / challenged — **scoped to the dive's own
   sheets** (`score_facts` gained `authoritative_sheets` so the NOI tab is the
   authority for cash flow, and `cells_block` so the challenge re-reads the
   dive's sheets); narrative finalized to verified facts; a Fact Check block is
   appended so credibility travels with the dive. Cost: 3 gpt-4o calls/dive
   (read → challenge → finalize), same as the brief, far less than the old
   agentic path.
